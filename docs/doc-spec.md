# Спецификация Требований (SRS)

## 1. Функциональные требования (FR)

### FR-01: Управление ордерами
*   **FR-01.1:** Система должна уметь открывать LONG позицию (покупка) по рыночной цене.
*   **FR-01.2:** Система должна уметь открывать SHORT позицию (продажа) по рыночной цене.
*   **FR-01.3:** Система должна уметь полностью закрывать позицию по указанному символу (обратная сделка на весь объем).

### FR-02: Информационные запросы
*   **FR-02.1:** Система должна запрашивать и отображать баланс USDT на фьючерсном кошельке.

### FR-03: Интерфейс
*   **FR-03.1:** Управление осуществляется через аргументы командной строки (CLI).

## 2. Нефункциональные требования (NFR)

*   **NFR-01: Надежность (Reliability).** Бот должен автоматически повторять запрос при сетевых ошибках (до 3 раз).
*   **NFR-02: Производительность (Performance).** Время инициализации скрипта не должно превышать 2 секунд.
*   **NFR-03: Безопасность (Security).** API ключи не должны передаваться через аргументы командной строки.
