# Оптимизация Производительности

## 1. Инициализация CCXT
*   Класс `ccxt.mexc` инициализируется один раз при запуске скрипта.
*   Используется Lazy Loading (ленивая загрузка) рынков: метод `load_markets()` вызывается только при необходимости валидации сложных символов (в текущей версии v1.0 пропущен для скорости — `create_order` быстрее вернет ошибку, чем мы будем загружать 2MB данных о рынках).

## 2. Сетевое взаимодействие
*   Используется синхронный режим `ccxt`. Для CLI-утилиты, выполняющей 1 действие за запуск, это оптимальнее, чем оверхед на запуск `asyncio` Event Loop.
*   Таймауты запросов настроены на 10 секунд (стандарт), чтобы не "висеть" бесконечно при плохой связи.

## 3. Ресурсы
*   Скрипт потребляет < 50MB RAM, что позволяет запускать его даже на микро-инстансах (AWS t2.micro) или Raspberry Pi.
