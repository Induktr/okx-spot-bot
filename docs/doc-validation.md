### 4. (Правила Валидации)
Система использует **двухуровневую валидацию**:

## Уровень 1: Валидация Аргументов (argparse)
Происходит до запуска основной логики.

1.  **Action:** Должен быть одним из списка: `['long', 'short', 'close', 'balance']`. Иначе: системная ошибка `invalid choice`.
2.  **Amount:**
    *   Обязателен для действий `long` и `short`.
    *   Должен быть числом (`float`).
3.  **Symbol:**
    *   Обязателен для действий `long`, `short`, `close`.
    *   Должен быть строкой.

## Уровень 2: Бизнес-Валидация (MEXCClient)
Происходит внутри логики бота.

1.  **Количество (Amount):** Должно быть больше 0.
2.  **Тип данных:** Приведение типов к формату, требуемому биржей (строки в float и обратно при необходимости).
